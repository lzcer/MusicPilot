# 1. MusicPilot 首个版本发布

MusicPilot 是面向自托管用户的音乐库自动化工具，用一个 Web 界面串联音乐发现、资源搜索、下载提交、文件整理、元数据补全、音乐库刷新和歌单同步等流程。

本版本适合已经部署音乐库、下载器和资源站点的用户。MusicPilot 不提供音乐资源和下载渠道，只负责连接用户自行配置的服务，减少重复搜索、手动下载、手动整理和手动刷新音乐库的操作。

## 1.1. 支持范围

- 媒体库：Navidrome。
- 下载器：qBittorrent。
- 站点：OpenCD、HDFans。

## 1.2. 主要功能

- 音乐搜索与站点搜索：支持先搜索音乐元数据，再基于元数据到站点搜索候选资源；支持站点并发控制、排除关键词、搜索结果去重，以及按艺术家、标题、专辑等信息辅助过滤。
- 下载任务管理：支持将选中资源提交到 qBittorrent，跟踪下载任务状态、查看下载明细、删除任务，并在下载完成后触发后续整理和音乐库刷新流程。
- 文件整理与元数据处理：支持源目录、映射目录、复制整理等模式；支持自动刮削、手动整理、歌词和标签写入，并记录每个文件的整理状态、失败原因和实际整理类型。
- 歌单管理：支持导入外部歌单并在本地管理歌单条目；支持根据歌单条目搜索、下载和匹配本地音乐库；支持将本地歌单同步到 Navidrome，并选择同步账号和公开状态。
- 音乐库与歌手库：支持扫描和展示音乐库歌曲；支持维护歌手库、别名和合并关系，用于提升中文名、英文名、别名之间的匹配准确性；支持刷新歌单与音乐库之间的匹配状态。
- 系统管理：支持站点、下载器、音乐库、通知和系统参数配置；支持日志查看、仪表盘统计和文件管理；支持通过 Docker 环境变量控制基础部署参数。

## 1.3. 部署方式

本版本提供 Docker 镜像，可在 NAS 或服务器上通过 Docker Compose 部署。默认使用 SQLite，适合单机和 NAS 场景；需要更高并发或独立数据库时，也可以配置 PostgreSQL 连接。

首次启动后，需要在 Web UI 中配置站点、下载器、音乐库、整理规则和通知渠道。
